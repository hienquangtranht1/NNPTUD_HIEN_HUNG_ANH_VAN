PHẦN 0. GIẢI ĐÁP THẮC MẮC CHUNG

❓ "12 model = 12 files trong folder schemas/ đúng không?"
✅ Đúng. MongoDB dùng Mongoose, mỗi file trong schemas/ là 1 bảng (collection) = 1 model. Nhóm 4 người, quy tắc: 8 model base + 4 thành viên mỗi người thêm 1 = 12 schemas files.

❓ "Dùng MVC = 0 điểm?"
✅ Bài yêu cầu RESTful API: BE chỉ trả JSON qua res.send(). Không được res.render('...'). FE là app riêng (Postman, React, HTML...) gọi API này.

❓ "Bỏ FE bị trừ điểm?"
⚠️ Mỗi thành viên không làm FE bị trừ 5 điểm.

PHẦN 2. QUY TẮC LOGIC CODE CỐT LÕI (BẮT BUỘC TUÂN THỦ)

⚠️ Tất cả thành viên phải tuân thủ các quy tắc sau, không được phá vỡ:
  • Xóa mềm (Soft Delete): KHÔNG dùng .findByIdAndDelete(). Phải dùng findByIdAndUpdate(id, { isDeleted: true }). Mọi schema đều có trường isDeleted: Boolean default false.
  • Lọc dữ liệu: Mọi câu query GET list đều thêm điều kiện { isDeleted: false }.
  • Timestamps: Mọi schema có { timestamps: true } để auto sinh createdAt, updatedAt.
  • Chỉ trả JSON: Dùng res.send(data) hoặc res.json(data) — tuyệt đối không dùng res.render().
  • Controllers: Chỉ có DUY NHẤT controllers/users.js. Logic CRUD bình thường viết thẳng trong routes/*.js.

PHẦN 4. CẤU HÌNH FILE app.js (THAY ĐỔI CHỈ 1 LẦN - NHÓM TRƯỞNG LÀM)

File app.js sinh ra mặc định cần được sửa lại để: Mount đủ các Routes và kết nối MongoDB database tên QLTASK:

var createError = require('http-errors');
var express = require('express');
var path = require('path');
var cookieParser = require('cookie-parser');
var logger = require('morgan');
let mongoose = require('mongoose');

var indexRouter = require('./routes/index');

var app = express();

// view engine setup (giữ lại để không lỗi)  
app.set('views', path.join(__dirname, 'views'));
app.set('view engine', 'ejs');

app.use(logger('dev'));
app.use(express.json());
app.use(express.urlencoded({ extended: false }));
app.use(cookieParser());
app.use(express.static(path.join(__dirname, 'public')));

// === MOUNT ROUTES (mỗi thành viên thêm route của mình vào đây) ===
app.use('/', indexRouter);
app.use('/api/v1/auth',          require('./routes/auth'));
app.use('/api/v1/users',         require('./routes/users'));
app.use('/api/v1/roles',         require('./routes/roles'));
app.use('/api/v1/departments',   require('./routes/departments'));
app.use('/api/v1/projects',      require('./routes/projects'));
app.use('/api/v1/categories',    require('./routes/categories'));
app.use('/api/v1/milestones',    require('./routes/milestones'));
app.use('/api/v1/tasks',         require('./routes/tasks'));
app.use('/api/v1/tags',          require('./routes/tags'));
app.use('/api/v1/taskHistories', require('./routes/taskHistories'));
app.use('/api/v1/comments',      require('./routes/comments'));
app.use('/api/v1/attachments',   require('./routes/attachments'));
app.use('/api/v1/notifications', require('./routes/notifications'));
app.use('/api/v1/upload',        require('./routes/uploads'));

// === KẾT NỐI MONGODB database QLTASK ===
mongoose.connect('mongodb://localhost:27017/QLTASK');
mongoose.connection.on('connected', function () {
    console.log("✅ Connected to MongoDB QLTASK");
});
mongoose.connection.on('disconnected', function () {
    console.log("❌ Disconnected from MongoDB");
});

// catch 404
app.use(function (req, res, next) {
    next(createError(404));
});

// error handler
app.use(function (err, req, res, next) {
    res.locals.message = err.message;
    res.locals.error = req.app.get('env') === 'development' ? err : {};
    res.status(err.status || 500);
    res.render('error');
});

module.exports = app;

PHẦN 5. PHÂN CÔNG CHI TIẾT TỪNG NGƯỜI (HIỂN / HƯNG / ANH / VÂN)

Mỗi thành viên tự làm phần mình: tạo file schema, file route, và test Postman 4 method (GET list, GET by ID, POST, PUT, DELETE).

🔴 HIỂN — Nhóm Trưởng | Core & Bảo mật Auth
📁 Schemas phụ trách (tạo trong schemas/):

📌 schemas/roles.js — Vai trò: ADMIN, MANAGER, MEMBER

📌 schemas/departments.js — Phòng ban: Phòng IT, Phòng Marketing...

📌 schemas/users.js — Người dùng hệ thống (có hash bcrypt)

  • 🔑 Route AUTH.JS — Xác thực người dùng
    POST /api/v1/auth/register → Đăng ký tài khoản mới (yêu cầu: username, password, email, roleId)
    POST /api/v1/auth/login → Đăng nhập, trả về JWT Token RS256
    GET /api/v1/auth/me → Xem thông tin người đang đăng nhập (cần Token)
    POST /api/v1/auth/logout → Đăng xuất, xóa cookie token
    POST /api/v1/auth/changepassword → Đổi mật khẩu (cần Token, validate oldPass & newPass)
    POST /api/v1/auth/forgotpassword → Gửi email link reset
    POST /api/v1/auth/resetpassword/:token → Đặt lại mật khẩu
  • 👥 Route USERS.JS — Quản lý người dùng
    GET /api/v1/users → Lấy danh sách user (chỉ ADMIN / MODERATOR mới xem được)
    GET /api/v1/users/:id → Xem chi tiết 1 user (chỉ ADMIN)
    POST /api/v1/users → Tạo mới user thủ công
    PUT /api/v1/users/:id → Cập nhật thông tin user (fullName, email, avatar...)
    DELETE /api/v1/users/:id → Xóa mềm user (isDeleted = true)
  • 🏷️ Route ROLES.JS — Quản lý vai trò
    GET /api/v1/roles → Lấy danh sách vai trò
    POST /api/v1/roles → Tạo vai trò mới (VD: ADMIN, MEMBER)
    PUT /api/v1/roles/:id → Sửa tên vai trò
    DELETE /api/v1/roles/:id → Xóa mềm vai trò
  • 🏢 Route DEPARTMENTS.JS — Quản lý phòng ban
    GET /api/v1/departments → Lấy danh sách phòng ban
    GET /api/v1/departments/:id → Xem chi tiết phòng ban (gồm danh sách user thuộc phòng đó)
    POST /api/v1/departments → Tạo phòng ban mới
    PUT /api/v1/departments/:id → Đổi tên hoặc mô tả phòng ban
    DELETE /api/v1/departments/:id → Xóa mềm phòng ban
  • 🔐 Utils phụ trách: Viết utils/authHandler.js (hàm CheckLogin xác thực JWT RS256, hàm CheckRole("ADMIN") chặn route), utils/mailHandler.js, utils/data.js, utils/validatorHandler.js

📁 Schemas phụ trách (tạo trong schemas/):

📌 schemas/projects.js — Dự án: tên, mô tả, trạng thái, ngày bắt đầu/kết thúc

📌 schemas/categories.js — Danh mục loại công việc: Tính năng, Lỗi, Cải tiến...

📌 schemas/milestones.js — Cột mốc giai đoạn: Sprint 1, Sprint 2, ngày deadline

  • 📂 Route PROJECTS.JS — Quản lý Dự án

    GET /api/v1/projects?status=active → Lọc dự án theo trạng thái (active/completed/pending)
    GET /api/v1/projects?name=abc → Tìm kiếm dự án theo tên (dùng RegExp giống mẫu thầy)
    GET /api/v1/projects/:id → Xem chi tiết 1 dự án (kèm thông tin phòng ban phụ trách)
    GET /api/v1/projects/:id/tasks → Lấy toàn bộ task thuộc dự án này
    POST /api/v1/projects → Tạo dự án mới (truyền name, departmentId, status, startDate, endDate)
    PUT /api/v1/projects/:id → Cập nhật thông tin dự án (VD: kéo dài deadline)
    DELETE /api/v1/projects/:id → Xóa mềm dự án
  • 🗂️ Route CATEGORIES.JS — Quản lý Danh mục loại CV
    GET /api/v1/categories → Lấy danh sách danh mục (có thể kèm ?name=loi để tìm kiếm)
    GET /api/v1/categories/:id → Xem chi tiết 1 danh mục
    GET /api/v1/categories/:id/tasks → Lấy tất cả task thuộc danh mục này
    POST /api/v1/categories → Tạo danh mục mới (name, slug auto gen từ slugify)
    PUT /api/v1/categories/:id → Cập nhật tên danh mục
    DELETE /api/v1/categories/:id → Xóa mềm danh mục
  • 🏁 Route MILESTONES.JS — Quản lý Cột mốc Sprint
    GET /api/v1/milestones → Lấy danh sách cột mốc
    GET /api/v1/milestones?projectId=xxx → Lọc cột mốc theo dự án
    POST /api/v1/milestones → Tạo cột mốc Sprint (name, projectId, dueDate, description)
    PUT /api/v1/milestones/:id → Cập nhật deadline hoặc tên sprint
    DELETE /api/v1/milestones/:id → Xóa mềm cột mốc
  • 🔧 Utils phụ trách: Viết, kiểm tra utils/IDHandler.js (nếu dùng custom ID)
🟢 ANH — Dev 3 | Lõi Quản lý Công việc (Task)
📁 Schemas phụ trách (tạo trong schemas/):

📌 schemas/tasks.js — Nhiệm vụ: tiêu đề, mô tả, người giao, người nhận, hạn nộp, trạng thái

📌 schemas/tags.js — Nhãn gán vào Task: Bug, Feature, Urgent, Hotfix

📌 schemas/taskHistories.js — Lịch sử mỗi lần đổi trạng thái task

  • ✅ Route TASKS.JS — Quản lý Nhiệm vụ Công việc
    GET /api/v1/tasks → Lấy danh sách tất cả task (isDeleted: false)
    GET /api/v1/tasks?status=todo → Lọc task theo trạng thái: todo / in-progress / done
    GET /api/v1/tasks?assigneeId=xxx → Xem task được giao cho ai
    GET /api/v1/tasks?projectId=xxx → Lấy task theo dự án
    GET /api/v1/tasks/:id → Xem chi tiết 1 task (có populate tên user nhận việc, tên dự án)
    POST /api/v1/tasks → Tạo nhiệm vụ mới (title, description, assigneeId, projectId, dueDate, status, categoryId)
    PUT /api/v1/tasks/:id → Cập nhật task: Đổi trạng thái (todo→in-progress→done), đổi người nhận...
    DELETE /api/v1/tasks/:id → Xóa mềm task
  • 🏷️ Route TAGS.JS — Quản lý Nhãn (Tag)
    GET /api/v1/tags → Lấy danh sách nhãn
    POST /api/v1/tags → Tạo nhãn mới (name: Bug, colorCode: #e74c3c)
    PUT /api/v1/tasks/:id/tags → Gắn/Gỡ nhãn cho 1 task
    DELETE /api/v1/tags/:id → Xóa mềm nhãn
  • 📜 Route TASKHISTORIES.JS — Lịch sử thay đổi Task (Chỉ xem)
    GET /api/v1/taskHistories?taskId=xxx → Xem toàn bộ lịch sử thay đổi trạng thái của 1 task
    → Logic: Khi route PUT /tasks/:id nhận status mới ≠ status cũ, tự động save() thêm 1 bản ghi vào taskHistories: {taskId, changedBy, oldStatus, newStatus, changedAt}
  • 🔧 Utils phụ trách: Hỗ trợ hoàn thiện utils/validatorHandler.js (validate khi tạo task)

📌 schemas/attachments.js — File đính kèm: filename, url, taskId, uploadedBy

📌 schemas/notifications.js — Thông báo: content, userId nhận, isRead, taskId liên quan

  • 💬 Route COMMENTS.JS — Bình luận trong Task
